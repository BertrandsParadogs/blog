    Design Pinterest

    # Design Pinterest

    # Requirements

    # Functional

    • User can view pins in the home page.
    • User can see more pins when they scrolls down.
    • User can click into a pin and view details about the pin.
    • User can search for relevant pins using a search bar.
    • User can like and save a pin.

    # Non-functional

    • The system should be scalable. As user base grows, the system should be able to handle increased load.
    • The system have low latency. The feed should load quickly for users.
    • The system should support dark mode and accessible to users with disability.
    • The system should be highly available, users should be able to access the feed at any time.

    # Tech stack

    • React for building fast and modular UI components.
    • Redux for state management.
    • GraphQL as API for querying the server for efficient data retrieval.

    # Architecture components

    # User interface

    • HomeFeed component
      • Display a grid of pins.
      • The container should be scrollable.
    • PinCard component
      • Individual pin card used to display either photo or video.
      • When hovered over, it will show like and save buttons.
      • When clicked, it will take the user to pin detail view.
    • PinDetailView component
      • Show the pin photo and video on the left side.
      • Show detail information about the pin, including author name, published time, text descriptions.
      • Show save and like button on the right side.
    • SearchBar component
      • Show input box where user can input search keyboards.
      • When there is input, show a dropdown suggestion list that expands on what user types.
      • When user press "Enter" key or click on any suggestions or click the search button, search query is performed.
      • Loading state shown while server is generating response.
      • Search result shown after receiving response from server.

    # Layout

    Pinterest layout is also called masonry layout. It is a grid layout that works by placing elements in optimal positions based on the available vertical space.

    • Pins are rendered into fixed-with columns.
    • Preserve the aspect-ratio of the image. 
      • img {
    aspect-ratio: 4 / 3;
    width: 100%;
    object-fit: cover;
}
    • Show the same image of different size for different viewport.
    • Pagination support with infinite scroll and dom virtualization.

    # Pin positioning

    We should use absolute position for the pins.

    Here is an algorithm for balancing the heights.

    function getShortestColumn(colHeights) {
    let shortestIndex = 0;
    let minHeight = colHeights[0];
    for (let i = 1; i < colHeights.length; i++) {
        if (colHeights[i] < colHeights[shortestIndex]) {
            shortestIndex = i;
            minHeight = colHeights[i];
        }
    }
    return shortestIndex;
}

    # Infinite scroll

    In traditional pagination, we organize documents by pages. Each page will hold a certain number of items and display them as the user decides to navigate forward and back, clicking either page number or the arrow icon.

    However, this approach requires manual user clicks to access more pages. With infinite scroll, we can solve this issue by dynamically appending the pages as the user scrolls, incrementing the page one at a time until we run through all the pages. With this approach, we could reduce interruption, lower the user interaction cost, and make it more user friendly.

    Drawbacks:

    1. Difficult to refind content since we don't have page number.
    2. Inability to access the footer.
    3. Increased page load. May affect experience of user with low network bandwidth.

    # Implementation using Intersection Observer

    The Intersection Observer API provides a way to asynchronously observe changes in the intersection of a target element with an ancestor element or with a top-level document's viewport.

    Intersection information is useful for:

    1. Lazy-loading of images or other content as a page is scrolled.
    2. Implement "infinite scrolling" website.
    3. Reporting of visibility of advertisements in order to calculate ad revenues.
    4. Deciding whether or not to perform tasks or animation processes based on whether or not the user will see the result.

    Here is how to create an interaction observer:

    let options = {
    root: document.querySelector("#scrollableArea"),
    rootMargin: "50px",
    threshold: 1.0,
};

let observer = new InteractionObserver(callback, options);

// callback
(entries) => {
    entires.forEach((entry) => {
        // Each entry describes an intersection change for one observed target element:
        //   entry.boundingClientRect
        //   entry.intersectionRatio 0.0 - 1.0
        //   entry.intersectionRect
        //   entry.isIntersecting bool
        //   entry.rootBounds
        //   entry.target
        //   entry.time
    });
};

let pin = document.createElement("div");
observer.observe(pin);
observer.unobserve(pin);
